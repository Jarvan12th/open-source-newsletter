# 每周巡检清单

## 一、信息雷达

- [ ] 回看本周 `01_信息雷达/01_每日扫描/`，筛出最重要的 5-10 条开源动态
- [ ] 检查 `01_信息雷达/05_监测规则/信源注册表.md`，确认高优先级信源没有漏看
- [ ] 补充本周新出现的项目名、组织名、许可证词、漏洞词和维护者姓名
- [ ] 单独复盘 GitHub Trending、Explore/Topics、Show HN、Reddit 自托管/开源讨论中的热门项目
- [ ] 标记持续发酵的话题，必要时放进 `03_热点追踪/`
- [ ] 回查上周是否漏掉后来快速升温的重要项目和事件
- [ ] 复核高热度事件是否同时具备足够可信度，避免把社区传闻当成事实
- [ ] 给热门项目补齐 `public_interest_score`、`practice_score`、demo/截图和 10 分钟实践路径

## 二、候选选题

- [ ] 从本周热点里挑出 3-5 个值得讲的话题
- [ ] 区分“快讯型更新”和“值得做观点型解读”的话题
- [ ] 给每个候选选题补齐发布说明、社区反应、实际影响和争议点
- [ ] 至少保留 1-2 个“容易理解、容易动手试”的项目型选题
- [ ] 写一句话判断：这件事为什么值得开源从业者或开发者关注

## 三、选题研判

- [ ] 在 `02_选题研判/周度综述/` 输出每周热点脉络摘要
- [ ] 在 `02_选题研判/` 记录背景、生态影响和潜在误判点
- [ ] 判断每个话题属于短期热度、长期趋势，还是结构性变化
- [ ] 提炼标题角度、讲述顺序和核心判断

## 四、内容生产

- [ ] 从周度摘要里挑出 3-5 个可做主题
- [ ] 产出提纲或脚本初稿
- [ ] 整理截图、release note、issue 讨论和必要配图
- [ ] 准备标题、简介和平台发布文案

## 五、发布复盘

- [ ] 记录已发布内容的点击、互动、收藏和评论反馈
- [ ] 总结哪些项目或议题真正打动受众，哪些只是一时热度
- [ ] 反向更新 `01_信息雷达/05_监测规则/` 的关键词和信源注册表
