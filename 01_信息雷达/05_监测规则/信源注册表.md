# 信源注册表

这是当前项目的 `source_tracker`，用于定义“看什么、多久看一次、优先级是什么”。

字段说明：

- `source_name`：信源名称
- `type`：官方源 / 平台源 / 媒体源 / 社区源 / 安全源 / 政策源
- `priority`：`P0` / `P1` / `P2`
- `coverage`：覆盖主题
- `check_frequency`：巡检频率
- `signal_to_noise`：信噪比，取 `高 / 中 / 低`
- `notes`：补充说明

## 使用规则

- 优先看项目官方仓库、release note、基金会博客和安全公告等一手源。
- 社区讨论负责提供“传播和争议信号”，不替代事实核实。
- 当同一事件同时出现在官方源、媒体和社区时，优先引用官方原始链接。
- 重点项目建议单独维护 watchlist，避免只依赖热榜。
- 对许可证、治理和安全类话题，至少做一次交叉验证后再进入高优先级判断。
- 对热榜型发现，除非能确认真实维护或采用信号，否则不要仅凭 star 增长就升级为重点事件。
