# 自动化巡检说明

这份文件定义 `automation_radar_rules`，用于把人工信息雷达拆成可自动执行、可复核、可进入每日候选池的步骤。

## 巡检目标

- 捕获开源生态中的重大变化：发布、安全、治理、许可证、维护状态。
- 捕获 GitHub 等平台上快速升温、容易理解、容易实践、适合大众内容化的开源项目。
- 每条候选都必须能回到原始链接，热门项目也必须补充维护信号和实践路径。

## 每日自动化任务

| task_id | frequency | input_sources | output | rule |
| --- | --- | --- | --- | --- |
| `github_trending_scan` | 每日 | GitHub Trending、GitHub Topics | 热门项目候选 | 记录仓库、语言、stars、描述、demo、最近提交 |
| `github_search_growth_scan` | 每日 | GitHub Search | 快速增长项目候选 | 查询近 30 天创建或活跃、stars 增长明显、topic 清晰的项目 |
| `show_hn_project_scan` | 每日 | Hacker News Show HN | 可试玩项目候选 | 只收录有 GitHub、demo、清楚问题陈述的项目 |
| `community_interest_scan` | 每日 | Reddit、Lobsters、X/Mastodon | 社区兴趣信号 | 标记讨论热度，不直接当作事实来源 |
| `official_release_scan` | 每日 | 官方博客、release note、基金会公告 | 重大动态候选 | 优先收录影响范围明确的发布、路线图和治理变化 |
| `security_policy_scan` | 每日 | GitHub Security Advisories、OSV、OpenSSF | 高优先级风险候选 | 安全、许可证、合规类事件单独标注影响范围 |

## 热门项目筛选规则

热门项目进入每日候选池前，需要同时满足至少三项：

- 有明确使用场景：一句话能说清楚它解决什么问题。
- 有热度证据：GitHub Trending、star 增长、Show HN 讨论、社区转发或媒体跟进。
- 有实践路径：README、quickstart、Docker Compose、在线 demo、playground 或可截图界面。
- 有维护信号：近期 commit、release、issue 响应、维护者说明或活跃讨论。
- 有内容价值：适合做“这是什么、为什么火、怎么试、适合谁”的短内容。

以下情况降级或丢弃：

- 只有 star 数，没有 demo、文档或近期维护信号。
- 项目描述过于抽象，普通读者难以理解使用场景。
- 安装路径高度复杂，短期无法验证。
- 许可证不清楚、仓库来源可疑、疑似营销刷榜。

## 自动化字段映射

写入 `每日候选池模板.md` 时，热门项目应补齐这些字段：

- `event_type`：优先用 `热门项目发现`、`项目发布`、`生态扩散`。
- `heat_score`：看传播速度和跨平台讨论。
- `public_interest_score`：看普通读者是否容易理解、是否有强场景。
- `practice_score`：看 10 分钟内能否跑通、试玩或截图。
- `project_discovery.popularity_evidence`：记录 Trending、Show HN、Reddit、star 增长等证据。
- `project_discovery.practice_path`：记录 quickstart、demo、Docker Compose、在线体验或最小实践步骤。
- `project_discovery.maintenance_signal`：记录最近提交、release、issue 响应或维护者声明。

## 推荐查询

- GitHub Trending：按 `daily`、`weekly`、主要语言和主题巡检。
- GitHub Search：`created:>YYYY-MM-DD stars:>500 pushed:>YYYY-MM-DD topic:<topic>`。
- GitHub Search：`stars:>1000 pushed:>YYYY-MM-DD "docker compose" "demo"`。
- Hacker News：`Show HN GitHub open source demo`。
- Reddit：`site:reddit.com/r/selfhosted GitHub open source`。
- Awesome Lists：`awesome <topic> GitHub`，用于补充同类项目对比。

## 输出节奏

- 每天至少记录 1 条“重大开源动态”或说明当天无高价值候选。
- 每天至少扫描 3 个热门项目入口，候选不达标也要在“今日归并说明”里简记原因。
- 每周从热门项目中挑 1-2 个进入 `本周热门项目发现`，优先选择能演示、能截图、能形成教程片段的项目。
