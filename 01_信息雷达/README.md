# 信息雷达说明

这一层只做一件事：稳定捕获开源领域的重要变化，并把“新闻线索”整理成“可判断、可追踪、可转化为选题”的信息。

建议优先关注以下六类信号：

- 重点项目的发布、版本更新、路线图和治理变化
- GitHub、GitLab、CNCF、Apache、Linux Foundation 等平台和组织动作
- 安全漏洞、供应链攻击、许可证调整和维护状态变化
- 明星仓库、工具链、框架和数据库的生态扩散
- GitHub 等平台上快速升温、容易讲清楚、容易动手试的开源项目
- 维护者发声、社区争议、基金会决策和商业支持动态

目录使用方式：

- `01_每日扫描`：按巡检日期记录每日候选池，允许合理时间偏差
- `02_重点信源`：长期维护固定监测入口
- `03_热点追踪`：给持续发酵的项目或事件做专题记录
- `04_候选选题`：把值得讲的新闻转成内容候选
- `05_监测规则`：维护雷达方法论、评分标准和信源注册表

这一层的标准产出只有三类：

- `每日候选池`：按当天巡检归档、允许补录近 1-3 天重要事件的候选集合
- `每周热点脉络摘要`：把一周持续发酵的话题聚合成主题线
- `高优先级突发清单`：需要快速响应的事件提醒
- `热门项目发现`：记录 GitHub、Hacker News、Reddit、Product Hunt 等平台上适合大众理解和开发者实操的新项目

统一评分维度：

- `heat_score`：讨论热度和传播速度
- `credibility_score`：信息源可信度和交叉验证程度
- `discussion_signal.score`：是否出现多平台扩散、维护者回应和二次解读
- `impact_scope`：对开发者、组织或生态的影响范围
- `public_interest_score`：大众是否容易理解、是否有明确使用场景、是否具备传播话题性
- `practice_score`：是否能在短时间内本地运行、在线体验或用截图演示
