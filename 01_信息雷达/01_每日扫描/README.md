# 每日扫描记录建议

建议按日期新建文件，例如：

- `2026-05-01_每日候选池.md`
- `2026-05-02_每日候选池.md`

每篇建议直接基于 `每日候选池模板.md` 创建，并保持统一字段，方便未来自动化写入。

这里的“每日”指的是“按当天巡检归档”，不是“事件必须发生在当天”。

允许的合理偏差包括：

- 前一天发布 release note，但当天社区讨论才明显升温
- 海外时区先发公告，次日才进入你的视野
- 周末或忙碌时漏扫，隔天补录仍重要的事件
- 同一项目在 24-72 小时内出现高价值后续进展

每日流程建议：

- 先看 `P0` 与 `P1` 信源
- 把候选事件写成标准事件卡片
- 允许补录近 1-3 天内仍有价值的事件，不必机械限制在自然日内
- 同一事件若有多篇报道，归并到一条卡片中
- 只在 `status=confirmed` 时把事实写死
- 对仅社区讨论或社媒爆出的消息，标记 `follow_up_needed=yes`

最小要求：

- 每条事件都要有原始链接
- 每条事件都要有 `heat_score` 和 `credibility_score`
- 每条事件都要区分“事件发生时间”和“本次巡检记录时间”
- 每日文件底部要列出“需明日跟进”的事件
