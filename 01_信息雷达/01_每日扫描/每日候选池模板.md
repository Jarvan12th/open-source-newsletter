## 评分说明

- `heat_score`
  - `1`：小范围讨论
  - `2`：垂直圈层讨论
  - `3`：多平台扩散
  - `4`：头部维护者、媒体和社区同步关注
  - `5`：行业级热点或生态广泛波及
- `credibility_score`
  - `1`：纯传闻，无有效来源
  - `2`：单一社媒来源，未交叉验证
  - `3`：有社区或媒体提及，但未见官方确认
  - `4`：多可信来源交叉验证
  - `5`：官方公告、仓库、release note 或公开材料可核实
- `discussion_signal.score`
  - `1`：仅单平台零散讨论
  - `2`：垂直社区有人提及，但未扩散
  - `3`：多平台开始出现转载或讨论
  - `4`：关键维护者发声，媒体或社区二次解读明显
  - `5`：官方源、开发者社区和社交平台同步扩散
- `impact_scope`
  - `project`：主要影响单个项目
  - `community`：影响一个社区或基金会
  - `ecosystem`：影响多个项目或工具链生态
  - `industry`：影响行业格局、政策或大范围采用
- `status`
  - `new`：新发现
  - `watching`：继续观察
  - `confirmed`：已确认
  - `shortlisted`：进入候选选题
  - `discarded`：确认不跟进
- `public_interest_score`
  - `1`：只有专业圈层能理解
  - `2`：需要较多背景知识
  - `3`：有明确使用场景，但需要解释
  - `4`：大众或普通开发者能快速理解价值
  - `5`：有强话题性、强可视化或强工具属性
- `practice_score`
  - `1`：难以复现，依赖复杂环境
  - `2`：可复现但门槛较高
  - `3`：有文档，可在半天内跑通
  - `4`：有 quickstart、demo、Docker Compose 或在线体验
  - `5`：10 分钟内可试玩、可截图、可形成教程片段
- `project_discovery.setup_difficulty`
  - `easy`：普通开发者可快速运行
  - `medium`：需要配置依赖、账号或 API key
  - `hard`：需要复杂环境、硬件或较深背景

时间字段说明：

- `event_date`：事件实际发生或首次公开的日期
- `record_date`：你本次巡检把它写入候选池的日期
- 如果两者不一致，不算异常；只要事件仍有跟进价值即可收录
